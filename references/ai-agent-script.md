# What is an AI Agent? — 3-Minute YouTube Explainer Script

## Frame 1: Hook (0:00–0:20)

**[TITLE CARD — dramatic intro]**

> "What if AI could do more than just answer your questions? What if it could actually *do things* for you — browse the web, write code, send emails, and complete entire projects while you sleep?"
>
> "That's not science fiction. That's an AI Agent. And by the end of this video, you'll understand exactly how they work."

**Visual:** Bold title "WHAT IS AN AI AGENT?" with orbiting icons (search, code, lightning bolt) around a glowing brain illustration.

---

## Frame 2: Problem (0:20–0:55)

**[Pain points — why traditional AI falls short]**

> "Here's the thing about regular AI — like a basic chatbot. You ask it a question, it gives you an answer. Done. It can't *do* anything."
>
> "It can't open your browser and research something. It can't check your email. It can't write code AND test it. It just... talks."
>
> "Traditional AI has three big limitations:
> 1. It answers questions but can't take action.
> 2. It forgets everything between conversations.
> 3. It can't use any tools — no web, no files, no APIs."

**Visual:** Split layout — frustrated user on left with X-marked bullet points; a wall labeled "LIMITATION" blocking them from a goal trophy.

---

## Frame 3: Concept (0:55–1:50)

**[Architecture — how AI agents actually work]**

> "An AI Agent fixes all of that. Here's how it works — three core components:"
>
> "First: **Memory.** An agent remembers what happened earlier in the task. It keeps context."
>
> "Second: **Reasoning.** At the core, there's a large language model — like Claude or GPT — that *plans* the next step. It doesn't just react. It strategizes."
>
> "Third: **Action.** The agent can actually *use tools*. It can search the web, run code, call APIs, read files."
>
> "And here's the magic: these three components run in a loop. The agent observes, plans, acts, then observes the result and plans again. Over and over until the task is done."

**Visual:** Central "AGENT CORE" diagram with three components (Memory, Reasoning, Action) connected by arrows, surrounded by tool icons (Web Search, Code Runner, API Calls), with a circular loop arrow labeled "OBSERVE → PLAN → ACT → REPEAT."

---

## Frame 4: Example (1:50–2:35)

**[Real-world walkthrough — research agent]**

> "Let me show you a real example. Say you tell an agent: 'Research the top 5 AI tools of 2025 and write me a report.'"
>
> "Here's what happens:"
>
> "Step 1: The agent understands the task. One instruction from you."
>
> "Step 2: It opens a web browser tool and searches Google. It does this six times, refining its queries."
>
> "Step 3: It reads 12 articles, pulls out the important data, and summarizes each one."
>
> "Step 4: It writes a structured, 5-page report and hands it to you."
>
> "You gave ONE instruction. The agent did 3 hours of research work in minutes."

**Visual:** Vertical 4-step flow with numbered nodes: (1) User gives instruction → (2) Agent searches web → (3) Agent reads & summarizes → (4) Agent writes report. Side callout: "Time saved: ~3 hours."
